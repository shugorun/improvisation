# MVP Technical（使い捨ての技術構成）

## 記録方法

- MVP を**最小コストで動かすために選んだ技術だけ**を書く。**設計は書かない**（本実装の設計・スタックは P3 `app-design/technical.md` → `04-ARCHITECTURE.md`）。
- これは**使い捨て**。本実装(P5)には引き継がない（本実装スタックは P3 で 0 から決める）。
- 実験で切り替えるバージョン（プロンプト / スキーマ / 検索設定）は `switches.md`。ここは土台の構成だけ。

## 構成

| 項目                            | 採用   | メモ                                                  |
| ------------------------------- | ------ | ----------------------------------------------------- |
| アプリ種別                      | ウェブ（SPA・完全クライアント） | 本実装も同種別の見込みだが、MVP の選定は引き継がない |
| 言語 / ランタイム               | TypeScript / Node v24 | ユーザ指定（React+TS）                                |
| 主要フレームワーク / ライブラリ | React 19 + Vite      | 最小コストで React+TS を即起動できる定番。状態管理は素の useState/useRef のみ |
| 起動 / 動かし方                 | `cd mvp && npm install && npm run dev`（Vite dev server） |                                 |

## 選定理由

- ユーザ指定の React+TS を**最小の足場**で動かせる組み合わせが Vite。中核体験（予測されて驚く）は純ロジック＋画面だけで検証でき、サーバ・DB・ルータ・状態管理ライブラリは不要。MVP は使い捨てなので lint/format/test は省略（手動確認で緑とみなす）。
