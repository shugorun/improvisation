## 記録方法

- MVP で**作る / 作らない**を書く。学びに応じて変わってよい。変えたら「変更ログ」に 1 行残す。
- これは本実装(P5)のスコープではない（それは P3 app-design / P4 spec）。

## 確かめたい問い

- ← / → を「ランダムに」押す人間を、単純な n-gram 予測器が 50% を有意に上回って当て続けられるか（＝中核体験「読まれて驚く」が成立するか）。
- どの予測モデル（頻度 / 1次マルコフ / n-gram）が最も"刺さる"手触りか。

## 作る（最小）

- ← / → 入力 → Oracle が押下前に予測 → ヒット/ミスを即時表示。
- ライブ精度表示（Oracle 的中率と試行数）。
- 予測モデルの**切替**（頻度 / 1次マルコフ / n-gram バックオフ）= 実験バージョン。

## 作らない（MVP では）

- 共有カード、モデル可視化、説明セクション、凝った演出、レスポンシブ最適化、i18n。
- これらは本実装(P5)で作る。MVP は中核体験の成立だけ確認する。
