## ビジョン

「自分は自由に・ランダムに選んでいる」という直感が錯覚であることを、説教でなく**1〜2 分の体験**で突きつける。
驚き → 新しい視点（自由意志・ランダムネス・情報理論）→ 人に見せたくなる、を一気に起こす。

## 解決する課題

- 人は自分の予測可能性を過小評価している。それを安全・短時間・無料で体験できる場が少ない。
- 既存の Aaronson Oracle 実装は学術デモ・素朴な作りが散在するだけで、**美しく・共有可能で・説明付きの製品**が存在しない。

## 対象ユーザ

| ユーザ | 状況 | 痛み | 代替手段 |
|---|---|---|---|
| シェアしたいネットユーザ | 面白い Web を探している | "すごい"と思える新ネタが少ない | 錯覚・心理テスト系サイト |
| 認知科学・確率に興味がある層 | 概念は知っているが体験がない | 「人はランダムになれない」を体感する手段がない | 論文・解説記事（読むだけ） |

## 価値仮説

- ← / → を押すだけの単純操作で、アプリが押下前に予測して当て続ける → 強い驚き。
- 予測精度・学習したパターン・「予測されやすさスコア」の可視化で、"なぜ当たるか"まで腑に落ちる。
- 終了時の共有カードと説明で、ブログ／SNS での紹介・拡散が自然に起きる。

## 非目標

今はやらないことを明確にする。

- アカウント登録・ログイン・サーバ保存（完全クライアントサイドで完結させる）。
- 課金・広告。
- ランダムネスの厳密な統計検定ツール化（あくまで"体験"が主役。説明は平易に留める）。

## 制約

- 技術制約: React + TypeScript（ユーザ指定）。完全クライアントサイド（外部 API・認証なし）。
- 時間制約: 個人開発・実験。MVP は使い捨てで素早く。
- 収益 / 配布制約: 静的ホスティングで無料公開できる形を志向。
- 個人開発として避けたい運用負荷: サーバ・DB・認証の運用は持たない。
