# ADR 0001: 本実装スタック — クライアントのみ React + Vite SPA

## ステータス

Accepted

## 背景

P3 で本実装(P5)のアプリ種別・言語・スタックを 0 から確定する必要がある（フェーズ移行ゲート）。
対象は Oracle: 完全にクライアントで完結する予測体験（ローカル計算・端末内保存で足りる）。言語は React + TypeScript がユーザ指定。

## 決定

本実装を**完全クライアントサイドの Web SPA**として構築し、以下のスタックを pin する。

- 種別: Web SPA（静的配信）
- 言語: TypeScript（strict）
- FW / ビルド: React 19 + Vite 6
- バックエンド: なし
- データベース: なし（永続化は localStorage のみ）
- 認証: なし
- デプロイ: 静的ホスティング（Vite build 成果物。配信先は初版で固定しない）
- 監視: なし（初版）
- テスト: Vitest（unit）+ tsc strict + ESLint + Prettier

## 根拠

- 体験が純クライアントで完結する（予測ロジックはローカル計算、保存はベストスコア程度）。サーバを持つ理由がない。
- サーバレスにより: 運用負荷ゼロ・無料の静的ホスティング・入力を外部送信しないプライバシー、を同時に得る。
- React+TS はユーザ指定。Vite は最小構成で速く、SPA を静的成果物として出せる。
- 予測ロジックの回帰防止が品質の要なので、domain を純関数化し Vitest で振る舞いテストする。
- 代替案: バックエンド付き（サーバ側リーダーボード等）→ 運用負荷とプライバシーのコストに見合う価値が初版にはない（非スコープ）。

## 影響

- ポジティブ: 運用ゼロ・無料配信・プライバシー・即起動・テスト容易（domain が純粋）。
- ネガティブ: サーバ前提の機能（共有リーダーボード、サーバ集計）は持てない（初版で非スコープなので許容）。
- 中立: 永続化は端末内のみ（クロスデバイス同期なし）。
