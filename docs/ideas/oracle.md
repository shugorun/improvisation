## 現在のアイデア

左右どちらかのキー（← / →）を「できるだけランダムに」押し続けるだけのゲーム。
ところがアプリ（Oracle）は、あなたが押す**前に**次を予測し、当て続ける。
人間は自分が思うほどランダムになれない —— 20 行程度の単純なアルゴリズム（Aaronson Oracle）が
あなたの「自由な」選択を 60〜70% 当ててしまう。「サイトに心を読まれた」体験を、
予測精度・学習したパターン・あなたの「予測されやすさスコア」の可視化とともに見せる。

## メイン仮説

- 「自分はランダムに振る舞える / 自由に選んでいる」という直感は錯覚で、行動には強いパターンがある。
- それを**体験**として突きつけると、強い驚き（すごい）＋ 新しい視点（自由意志・ランダムネス）＋ 拡散したくなる感情が同時に起きる。

## 対象ユーザ

- ネットで「面白い・賢い」ものを見つけて人にシェアしたい人（ブログ / SNS 読者）。
- 認知科学・確率・情報理論に軽い興味がある人。専門知識は不要。

## 課題（解く"痛み"ではなく、与える"気づき"）

- 人は自分の予測可能性を過小評価している。これを安全・短時間・無料で体験できる場が少ない。
- 既存の「Aaronson Oracle」は学術デモ・素朴な実装が散在するだけで、**美しく・共有可能で・説明付きの製品**がない。

## 解決仮説（＝体験設計）

- ← / → を押すだけ。Oracle が押下前に予測 → ヒット/ミスを即座に開示。
- ライブ HUD: Oracle 精度 vs 50% ベースライン、連続的中ストリーク、あなたの「予測されやすさ %」。
- 可視化: モデルが学んだ遷移確率（例「直近が R,L のときあなたは 68% で L」）を見せ、"なぜ当たるか"を腑に落とす。
- 終了時に共有カード（「Oracle は私を 64% 当てた。あなたは勝てる?」）。
- 説明セクション: Aaronson Oracle / Shannon の読心機 / なぜ人はランダムになれないか。

## 使われる場面

- ブログ / SNS で紹介 → クリック → 1〜2 分遊ぶ → 「勝てなかった」「読まれた」とシェア。
- 「ランダムに押してるのに当てられる」を友人に見せて驚かせる対面デモ。

## 代替手段 / 競合

- 散在する Aaronson Oracle デモ（素朴・説明やビジュアル・共有性に乏しい）。
- 各種「錯覚・認知バイアス」系サイト（後述の比較参照）。差別化は「単一の強い驚き × 美しさ × 共有性 × 説明」。

## MVP 候補（P2 / 使い捨て・`mvp/`）

- ← / → 入力 → Oracle 予測 → ヒット/ミス表示、精度ライブ表示。
- 予測モデルを**切替可能**にする（実験バージョン = このアプリの「切替したい設定」）:
  - v0: 頻度のみ（直近の L/R 比）
  - v1: 1 次マルコフ（直近 1 手から）
  - v2: n-gram（直近 k 手のパターン、バックオフ付き）— Aaronson 本来の形
- 目的: 「どのモデルがどれだけ当てるか」「中核体験＝"読まれて驚く"が成立するか」を確認。

## 検証方法

- 自分（製作者）が本気でランダムに押しても Oracle が 50% を有意に上回って当て続けることを確認。
- 「読まれた」という驚きが体験として成立するか（手触り）。
