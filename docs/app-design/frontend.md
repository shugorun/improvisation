# Frontend Design

## 記録方法

- 画面、ナビゲーション、UI 状態、入力、フィードバックを書く。
- 見た目の細部より、ユーザがどう操作するかを優先する。
- フロント方針の変更は「更新ログ」に日時付きで残す。

## 画面一覧

単一ページ（SPA・1 画面）にセクションを縦積みする。ルーティングは持たない。

| セクション | 目的 | 主な操作 | status |
|---|---|---|---|
| ヒーロー / コアゲーム | 操作即理解＋中核体験 | ← / → キー・タップ | active |
| ライブ統計 + 精度チャート | 「50% を超えて当てている」を可視化 | なし（自動更新） | active |
| 予測の中身（Insight） | なぜ当たるかを理解 | 折りたたみ開閉 | active |
| 共有 | スコアを人に見せる | 画像保存 / テキストコピー | active |
| 説明（How it works） | 知的な裏付け・ブログ素材 | 折りたたみ開閉 | active |
| 設定（上級） | モデル / order 切替・リセット | 選択・クリック | active |

## ナビゲーション

```text
1 画面スクロール。ファーストビュー = コアゲーム（操作説明は最小の 1 行）。
下にスクロール → 統計 / Insight / 共有 / 説明 / 設定。
```

## 主要画面

### コアゲーム（ファーストビュー）

- 目的: 説明を読まずに 30 秒で「読まれた」と気づかせる。
- 表示: タイトル＋一言、左右の大きな2面（← LEFT / RIGHT →）、直前ラウンドのフィードバック（「Oracle は LEFT を予測 — 捕捉！/ 出し抜いた！」）、直近手の strip（赤=捕捉/緑=回避）。
- 操作: キーボード ← / →、面のタップ/クリック。
- 空状態: 「ランダムに押してみて。次の一手を当てる」。最初の数手は「学習中…」。
- エラー状態: なし（入力は2択のみ）。

### ライブ統計 + 精度チャート

- 表示: Oracle 的中率（大）、ラウンド数、捕捉ストリーク、予測されやすさスコア、的中率推移の SVG ライン（50% 基準線つき）。
- 「学習中」フェーズ（< 一定ラウンド）はスコアを「測定中」にする。

### 予測の中身（Insight）

- 表示: 現在の文脈（直近 k 手）と Oracle の次予測＋信頼度、よく当てる文脈の上位（例「L,R,L → R 73%」）。
- 目的: 「あなたの癖はここ」と見せ、aha を作る。

### 共有

- 操作: 「結果を画像で保存」「テキストをコピー」。
- 出力: スコア＋一言（例「The Oracle predicted me 64% of the time. Can you beat it?」）＋アプリ名。完全クライアント生成。

### 説明（How it works）

- Aaronson Oracle / Shannon の読心機 / なぜ人はランダムになれないか、を平易に。折りたたみ既定で閉。

## UI 状態

- loading: なし（静的・即起動）。フォント/初期描画のみ。
- empty: ゲーム未プレイ（「最初の一手を押して」）。
- error: localStorage 不可時もサイレントに継続（機能は止めない）。
- success: 各ラウンドのフィードバックと統計更新。

## フォーム / 入力

- 主入力は ← / → キーとタップのみ。テキスト入力なし。
- 設定はボタン/トグル（モデル選択・order・リセット）。

## アクセシビリティ

- キーボード完結（← / → でプレイ、Tab で各操作に到達）。
- フィードバックは `aria-live="polite"` で読み上げ。
- 2面ボタンに明示ラベル（LEFT / RIGHT）。色だけに依存しない（捕捉/回避は記号＋色）。
- `prefers-reduced-motion` で演出アニメを抑制。
- 十分なコントラスト（ダーク基調＋アクセント）。
